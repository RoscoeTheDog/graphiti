## Sprint Goal

Improve the Graphiti MCP server user experience by making the daemon service auto-install and auto-enable, eliminating the silent connection failure that occurs when users configure Claude Code for HTTP transport but the daemon isn't running. Users should be able to run a single `daemon install` command and have everything work immediately.

## Context

**Problem**: Users configure Claude Code MCP server for HTTP transport (port 8321), but get silent "connection refused" errors because the daemon isn't running. There's no feedback to help users understand what's wrong.

**Solution**:
1. Auto-enable daemon after install (no manual config edit needed)
2. Add clear error messages when connection fails
3. Update documentation to reflect simplified flow
